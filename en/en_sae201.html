Action plan
  1. Analyse the assignment and understand the network topology to build in GNS3.
  2. Divide the 192.168.64.0/20 network into suitable subnets using variable-length subnet masks.
  3. Create VLANs 100, 200 and 300 to separate the different user groups.
  4. Configure inter-VLAN routing on the central switch.
  5. Set up dynamic RIP v2 routing between routers.
  6. Deploy network services: DHCP, FTP, Apache web server and synchronisation with rsync.
  7. Add security rules: port security on switches and ACLs on routers.
  8. Test connectivity between workstations, servers, VLANs and the external network.
  9. Correct configuration errors and verify the overall operation of the network.

Project summary for the portfolio

Context & objective

The objective of this SAE was to build a complete local network using GNS3. The project consisted of setting up an infrastructure composed of switches, routers, several VLANs and different servers. It was also necessary to configure network services such as DHCP, FTP and a web server, then add simple security rules to control access and protect the equipment.

Process & methodology

I began by analysing the required topology and preparing the IP addressing plan with subnetting. Then, I created VLANs 100, 200 and 300 to separate the user groups: administration, developers and administrators. I configured inter-VLAN routing, RIP v2 between routers, then DHCP, FTP and Apache services. Finally, I added security rules with ACLs and port security before testing connectivity between workstations, servers and the external network.

Feedback

What I learned

This SAE allowed me to better understand how to build a complete local network. I learned how to organise a network with several VLANs, set up inter-VLAN routing and use a dynamic routing protocol such as RIP v2. I also strengthened my skills in network services, especially DHCP, FTP, Apache and rsync. Finally, setting up ACLs and port security helped me better understand initial network security measures.

What I would do differently

With hindsight, I would prepare a more detailed addressing plan before starting the configurations. This would help avoid subnet or gateway errors. I would also test each step more gradually: first the VLANs, then routing, then services and finally security. This method would make troubleshooting easier in case of connectivity problems.
