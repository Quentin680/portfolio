Project summary for the portfolio

Context & objective

The objective of this project was to develop a reporting tool capable of analysing a file tree in order to identify the largest files on a disk. This issue is frequently encountered during computer maintenance when storage space becomes insufficient. The application had to locate the 100 largest files, represent them graphically and make it easier to delete them safely.

Process & methodology

The project was divided into several Python scripts. A first program allowed the user to choose the directory to analyse using PyQt5. A second script recursively scanned all subdirectories to retrieve full paths and file sizes. The results were then sorted and stored in a JSON file. Finally, a third script created a graphical interface made up of tabs, a pie chart and legend lists allowing certain files to be selected. A PowerShell script could then be generated to delete the chosen files.

Feedback

What I learned

This project allowed me to learn how to process large amounts of data, handle JSON files, recursively scan a folder tree, design a graphical interface in Python and automate certain administration tasks using PowerShell. I also discovered how to design a reporting tool and organise a project made up of several complementary scripts.

What I would do differently

With hindsight, I could have spent more time on the design phase in order to better structure the different scripts and anticipate certain difficulties related to file handling and the graphical interface. I could also have improved code readability by documenting it more and adding more checks to make the application more robust against errors. Finally, I could have optimised certain parts of the program to improve performance when analysing very large file trees.
