    Project summary for the portfolio

    Context & objective

    As part of SAE 1.02, I simulated the network infrastructure of a branch of the company RTC (Rudy and Theodor Company). The objective was to design, configure and test a local network including several VLANs, a router, a switch, a DHCP server and client workstations in order to ensure communication between the different departments of the company.

    Process & methodology

    The project began with an analysis of the requirements document in order to identify the company’s needs and the services to be implemented. I then designed the network architecture by defining the different VLANs and the associated IP addressing plan. Once the design was validated, I created the infrastructure in EVE-NG by adding the necessary equipment (router, switch, server and client workstations). I then configured the VLANs, inter-VLAN routing and the DHCP service to allow communication between the different networks. Finally, I carried out several connectivity and functionality tests to verify that all equipment and services met the requirements.

    Feedback

    What I learned

    This project allowed me to acquire the basics of network administration in a Cisco environment. I learned how to design a network architecture, segment a network using VLANs, create an IP addressing plan adapted to the needs of a company and configure inter-VLAN routing. I also discovered how DHCP services work and how to use network simulation and analysis tools such as EVE-NG and Wireshark to verify that the infrastructure operates correctly.

    What I would do differently

    With hindsight, I would spend more time on the design phase before starting to configure the equipment. I would define the VLAN organisation, IP addressing plan and needs of each service more precisely in order to have an overall view of the network from the start.
