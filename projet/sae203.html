Gestionnaire de notes

Développement en groupe de 3 d’une application web de gestion de notes, permettant de consulter, ajouter, modifier et supprimer des étudiants, des UE, des ressources, des enseignants, des examens et des notes. Le projet repose sur une base de données et une application déployée sur une machine virtuelle Linux.

Synthèse du projet pour le portfolio

Contexte & objectif

Cette SAE avait pour objectif de concevoir et déployer une application web permettant de consulter, insérer et manipuler des données à partir d’une base de données. Notre sujet était la création d’un gestionnaire de notes pour des étudiants. Le site devait permettre de gérer les étudiants, les enseignants, les UE, les ressources, les examens et les notes grâce à des fonctionnalités CRUD. Le projet a été réalisé en groupe de 3 étudiants : Quentin Altinok, Alexandre Kientz et Illia Sitovskyi.

Déroulement & méthodologie

Nous avons commencé par analyser le sujet et définir les données à gérer dans l’application : étudiants, UE, ressources, enseignants, examens et notes. Ensuite, nous avons organisé le travail avec un diagramme de Gantt afin de répartir les tâches entre les membres du groupe. Nous avons mis en place le projet Django, conçu le schéma de base de données, puis développé les différentes fonctionnalités CRUD. Enfin, nous avons ajouté les fonctions importantes du projet : connexion enseignant, ajout manuel des notes, import par fichier Excel, consultation des notes et génération d’un relevé de notes en PDF.

Retour d'expérience

Ce que j'ai appris

Cette SAE m’a permis de progresser en développement web et en gestion de base de données. J’ai appris à utiliser Django pour créer une application structurée, à manipuler des données liées aux étudiants, aux UE, aux examens et aux notes, ainsi qu’à mettre en place un CRUD complet. J’ai aussi découvert l’importance de l’authentification, de l’organisation des templates et de la séparation entre les modèles, les vues, les formulaires et les routes. Le travail en groupe de 3 m’a également permis de mieux comprendre l’importance de la répartition des tâches, de la communication et de l’utilisation d’un dépôt GitHub commun.

Ce que je ferais différemment

Avec du recul, je préparerais plus précisément la structure de la base de données avant de commencer le développement afin d’éviter certaines modifications en cours de projet. J’organiserais aussi les tâches plus tôt dans le groupe, avec des objectifs plus précis pour chaque séance. Enfin, je mettrais en place des tests plus réguliers après chaque fonctionnalité pour vérifier rapidement les formulaires, les imports de fichiers et l’affichage des relevés de notes.
